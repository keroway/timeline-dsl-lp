まず短い `.tdsl` を読み、Playgroundで試し、最後にCLIで保存・検証します。

## 1. 読む

```txt
timeline "読書メモ" {
    title "読書メモ";
    unit year;
    range 2024..2026;
    calendar proleptic_gregorian;
}

lane "本" as books { kind custom; order 10; }

event books 2025 "Timeline DSLを試す" { id "event:first"; };
```

`timeline` で年表全体、`lane` で縦軸、`event` で出来事を書きます。文法の読み方は [文法とサンプル](/docs/grammar/) に短い例をまとめています。

## 2. Playgroundで試す

[このサンプルをPlaygroundで開く](/playground/?source=timeline+%22%E8%AA%AD%E6%9B%B8%E3%83%A1%E3%83%A2%22+%7B%0A++++title+%22%E8%AA%AD%E6%9B%B8%E3%83%A1%E3%83%A2%22%3B%0A++++unit+year%3B%0A++++range+2024..2026%3B%0A++++calendar+proleptic_gregorian%3B%0A%7D%0A%0Alane+%22%E6%9C%AC%22+as+books+%7B+kind+custom%3B+order+10%3B+%7D%0A%0Aevent+books+2025+%22Timeline+DSL%E3%82%92%E8%A9%A6%E3%81%99%22+%7B+id+%22event%3Afirst%22%3B+%7D%3B)

Playgroundでは、`source` クエリで渡した本文がそのままエディタに入り、ブラウザ内で検証とSVGプレビューを確認できます。

## 3. CLIで保存して検証する

同じ内容を `sample.tdsl` として保存し、CLIで検証します。

```sh
tdsl check sample.tdsl
tdsl lint sample.tdsl
```

出力が成功すれば、構文と基本的な品質チェックを通過しています。JSONとして確認する場合は `build` を使います。

```sh
tdsl build sample.tdsl --pretty --output sample.json
```

ブラウザで確認する場合は、スタンドアロンHTMLとして描画します。

```sh
tdsl render sample.tdsl --output sample.html
```
