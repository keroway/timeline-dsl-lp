## `tdsl` が見つからない

インストール後に `tdsl --help` が実行できない場合は、インストール先が `PATH` に含まれているか確認します。

```sh
which tdsl
tdsl --help
```

インストールスクリプトを使った場合、既定の配置先は `~/.local/bin/tdsl` です。

```sh
export PATH="${HOME}/.local/bin:${PATH}"
```

## `.tdsl` の構文エラーを切り分けたい

まず `check` で構文・意味チェックを実行します。パース結果を確認したい場合は `ast` を使います。

```sh
tdsl check sample.tdsl
tdsl ast sample.tdsl
```

## lane参照やIDの問題を直したい

`lint` は未定義lane参照、重複id、`start > end`、空label、タグの空要素・重複などを検出します。安全に直せる項目は `--fix` で修正できます。

```sh
tdsl lint sample.tdsl
tdsl lint sample.tdsl --fix
```

CIや別ツールで扱う場合はJSON形式にします。

```sh
tdsl lint sample.tdsl --format json
```

## Wikidata取得が不安定

Wikidata import を含むファイルでネットワークやキャッシュの影響を避けたい場合は、`--offline` で静的項目だけを処理します。

```sh
tdsl build sample.tdsl --offline --pretty
tdsl render sample.tdsl --offline --output sample.html
```

最新データを取り直したい場合は `--no-cache`、キャッシュTTLを変えたい場合は `--cache-ttl <秒>` を指定します。

```sh
tdsl build sample.tdsl --no-cache --pretty
tdsl build sample.tdsl --cache-ttl 0 --pretty
```

## HTMLの見た目を調整したい

`render` の表示オプションを調整します。

```sh
tdsl render sample.tdsl \
  --output sample.html \
  --scale 5 \
  --lane-height 80 \
  --theme dark
```

独自CSSを追加する場合は `--custom-css` を使います。

## `timeline.range` 外のアイテムで警告が出る

v1.23.0 以降、`event` / `span` / `event_range` が `timeline.range` の外にある場合、`tdsl check` / `tdsl build` / LSP が警告を出すようになりました（従来は無診断でRendererが黙ってドロップ/クリップしていました）。

- `Event "{id}" at {time} is outside timeline.range and will not be rendered`
- `{Span|EventRange} "{id}" is entirely outside timeline.range and will not be rendered`
- `{Span|EventRange} "{id}" is partially outside timeline.range and will be clipped`

意図的に表示範囲を絞っている場合は無視して構いません。タイプミスであれば `timeline.range` またはアイテムの日付を見直してください。

## 未知の `unit` / `kind` でエラー・警告が出る

v1.23.0 以降、`timeline.unit` に未知の値（`year` / `month` / `day` / `hour` / `minute` 以外）を指定するとエラーになります（従来は無診断で `year` にフォールバックしていました）。`lane` の `kind` に未知の値を指定した場合は警告（W204）になりますが、`kind` は自由分類のためビルドは継続します。

```sh
tdsl check sample.tdsl
```

`unit` のタイプミスは `year` / `month` / `day` / `hour` / `minute` のいずれかに修正してください。`kind` は独自分類であることを明示したい場合 `kind custom;` を使います。

## Playground で WASM が読み込めない

[Playground](/docs/playground/) はブラウザ内で WASM を実行します。「Loading…」のまま動かない、または描画されない場合は、まずブラウザの開発者ツールの Console / Network タブを開きます。

- WASM ファイルのリクエストが `200` で返り、レスポンスの `Content-Type` が `application/wasm` になっているか確認します。`application/octet-stream` などで配信されているとブラウザが streaming compile に失敗します。
- 社内プロキシや広告ブロッカーが `.wasm` のリクエストを遮断していないか確認します。
- Console に出ているエラーメッセージ（`CompileError` / `LinkError` など）をそのまま確認し、ネットワーク要因か実行時要因かを切り分けます。

## Playground の表示が古い・反映されない

デプロイ直後に古い結果が表示される場合は、キャッシュが原因です。

- ハードリロード（macOS: `Cmd+Shift+R` / Windows: `Ctrl+Shift+R`）でブラウザキャッシュを無視して再取得します。
- それでも直らない場合は、開発者ツールの Application タブから当該サイトの Cache Storage / Service Worker をクリアします。
- 新しい DSL 構文を試していて parse エラーになる場合は、Playground が読み込む WASM が古い可能性があります。最新のデプロイが反映されているかを確認してください。

## 非対応ブラウザ

Playground は [WebAssembly](https://developer.mozilla.org/ja/docs/WebAssembly) を必須とします。モダンな Chrome / Edge / Firefox / Safari では動作しますが、WebAssembly を無効化した環境や、極端に古いブラウザでは動きません。

- ブラウザを最新版に更新するか、別のモダンブラウザで開き直します。
- ブラウザ拡張で WebAssembly を無効化している場合は解除します。
- 手元で確実に動かしたい場合は、ブラウザに依存しない [CLI](/docs/installation/)（`tdsl render`）の利用を検討してください。
