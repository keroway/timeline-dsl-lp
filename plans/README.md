## Findings considered and rejected/deferred

- **No security plan**: `pnpm audit --audit-level high` found no known vulnerabilities; no tracked credentials or evidence-backed exploitable sink was found. SVG/release HTML sinks are derived from vendored/build-time trusted pipelines and sanitized paths; do not label them XSS without a demonstrated trust-boundary failure.
- **No wholesale redesign**: LP hierarchy, CTA structure, responsive components, dark/high-contrast tokens and 42-route axe results are strong and align with `DESIGN.md`. A redesign would have poor leverage.
- **No generic dependency-upgrade plan**: current core versions build/test cleanly; major updates alone are not evidence of user impact. Only the reproduced deprecation path merits investigation.
- **No test-coverage-percentage target**: critical behavior, not arbitrary percentage, is the gap. Pan/zoom and visual enforcement were selected.
- **No immediate performance rewrite**: bundle budgets pass (WASM 652,559 bytes; total JS 354,780; max JS 297,106) and no algorithmic hotspot was confirmed. WASM is close to its warning budget, so monitor release syncs rather than optimize speculatively.
- **No forced removal of maintainer docs**: Plan 001 first tries clear IA/LLM exclusion while preserving stable URLs. A new collection/migration is only justified if that cannot satisfy the contract.
